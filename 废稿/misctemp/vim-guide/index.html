    一些注意：¶

    配置 ~/.vimrc 的时候，直接 :wq 就可以了。不要 source ~/.vimrc ，会出事。

    Ctrl+w+hjkl 可以切换窗口

    :! Command 方式写下的命令，会在终端里运行 $ Command。

    Normal模式下按 i 进入插入模式，v 进入可视模式。

    把主题的 .vim 文件存在 ~/.vim/colors 下。

    把插件存在 ~/.vim/bundle 的对应文件夹下。

    比如 auto-pairs 存在 ~/.vim/bundle/auto-pairs

    最开始使用插件的时候要手动 git clone 插件管理器 Vundle 到 ~/.vim/bundle/Vundle.vim。

    之后装插件也可以直接这么干然后在 vimrc 里面加上插件名字。

    1
    git clone https://github.com/VundleVim/Vundle.vim.git ~/.vim/bundle/Vundle.vim

    在切换主题之后如果发现透明没了，在 colorscheme xxx 前加上 autocmd vimenter * hi Normal guibg=NONE ctermbg=NONE " transparent bg

    ·在 vimrc 里面写完了 Plugins

    之后，要 :PluginInstall。

    如果需要把系统剪贴板 +y/+p和 Vim y/p互通，

    加上 set clipboard^=unnamed,unnamedplus

    • 标签页

    • 打开文件在新标签页 :tabe filename :tabnew filename :tab split 在新标签页，打开当前缓冲区中的文件

    • 显示 :tabs 显示已打开标签页的列表,并用“>”标识出当前页面，用“+”标识出已更改的页面(即还未保存）。

    • 关闭 :tabc 关闭当前标签页 :tabo 关闭自己之外所有标签页(other)

    • 切换标签页 :tabn 或 gt 移动到下一个标签页 :tabp 或 gT 移动到上一个标签页 :tabfirst 或 :tabr 移动到第一个标签页 :tablast 移动到最后一个标签页

    • 移动标签页 :tabm number 请注意，标签页次序是从0开始计数的。比如命令:tabm 1将把当前标签页移动到第2的位置。如果你没有为:tabm命令指定参数，那么当前标签页将会被移动到最后。

    • 多标签命令 :tabdo 命令 可以同时在多个标签页中执行命令 比如我们打开了多个标签页，需要把这些文件中的“python”都替换成“mypython”，那么就可以使用:tabdo %s/python/mypython/g命令，一次完成对所有文件的替换操作，而不用针对每个文件重复操作。

    • 配置标签页 vimrc中设置 set showtabline=0 始终不显示标签页 set showtabline=1 新建了标签页才会在窗口上方显示标签栏 set showtabline=2 总是显示标签栏

    Vim默认最多只能打开10个标签页，你可以用set tabpagemax=15改变这个限制。

    标签页部分原文链接：https://blog.csdn.net/greybeard/article/details/84222045

    • 多行注释

    首先进入 Vblock 模式。选中所有的注释部分，然后 Shift+i ，·输入 // 然后按ESC

    在装了 NerdCommenter 之后可以直接 Vline 选中然后 \ci

    • 取消

    进入 Vblock ，只选中注释条 // 部分，按 d。

    当然如果直接用 Vline 也是可以的，就不用注意太多

    如果需要卸载用 .vmb 安装的插件，在.vim 目录下删除对应文件即可。

    term 可以打开终端。

    在 nerdtree 当中， r 可刷新当前目录。

    1. 移动窗口本身（轮换或交换）¶

    Ctrl + w + r：向右或向下方交换窗口，而Ctrl + w + R则和它方向相反。

    光标会随着窗口而移动

    Ctrl + w + x：交换同列或同行的窗口的位置。vim默认交换当前窗口的与它下一个窗口的位置，如果下方没有窗口，则试着与上一个窗口交换位置。亦可在此命令前加上数量，与制定的窗口交换位置。

    2. 移动窗口并改变其布局¶

    注：下面的第三个字母都是大写，按 Ctrl + w之后，按shif + 相应的字母。后面用^代替Ctrl Ctrl + w + K ：移动当前窗口至屏幕顶端，并占用全部宽度 ^wJ： 移动窗口至屏幕底端，并占用全部宽度 ^wH：移动窗口至屏幕左端，并占用全部高度 ^wL：移动窗口至屏幕右端，并占用全部高度

    ^wT ：移动窗口至屏新的现有分页

    Commenter :\ci = Ctrl+/

    常用操作¶

    • In Normal

    u ：等于 Ctrl+Z

    Ctrl+r:等于 Ctrl+Y

    Ctrl+s 锁定 vim。

    但是快捷键按了，在 Ctrlq之后就会全部dui上去。

    Ctrl+q 解除锁定。

    从普通模式进入插入模式有如下数个键位可选：

    • a：往后挪一个字符插入文本。
    • A：移动到当前行尾插入文本。
    • i：在光标当前位置插入文本。
    • I：移动到当前行头插入文本。
    • o：在当前行的下一行新建一行，并插入文本。

    • O：在当前行的上一行新建一行，并插入文本。

    • 返回普通模式的键位是Esc键；亦可使用快捷键Ctrl+[。

    • cd （没有 !） ：切换工作目录
    • pwd ：展示当前工作目录。

    v ：Visual：如果跨行，只能整选，但在行内可以划分

    V (Shift+v)：Visual Line ：只能整行整行选。

    y ：Copy

    h,j,k,l 左下上右 (方向键也是可以的)

    Ctrl+v:Visual Block 如果跨行，可以分割选，选取的时候，选取矩阵的某个角如果缺了，没有事情，那一块也能选入。

    在插入模式下，按 Ctrl+o 即可进入插入 - 普通模式，执行完一次操作后又会自动回到插入模式。

    dd 可以删除(cut)一整行；yy 可以复制当前行。

    假设光标在 line1，3dd 则会删除 line 1,2,3

    gg 命令可跳至代码的开头；G 命令可跳至代码最后一行；G 命令前加数字可跳至指定行。

    :12/12G 会跳转到 Line 12。

    普通模式下按/，下方即会出现查找框，输入需要查找的字符，按回车后就能查看搜索结果。如果有多个查找结果，按n即可跳至下一个查找结果；按N可跳至上一个。

    file xxx 把当前文件命名为xxx。

    x 删除当前光标位置的字符

    dw 删除光标所处的单词

    d$ 删除当前行

    d motion:

    1
2
3
      w - 从当前光标当前位置直到下一个单词起始处，不包括它的第一个字符。
  e - 从当前光标当前位置直到单词末尾，包括最后一个字符。
  $ - 从当前光标当前位置直到当前行末。

    这些 motion 也可以在移动，选择，修改当中作为后缀。

    在命令之前添加数字可以让它重复那么多次。

    dd 删除整行

    % 括号匹配。

    / xxx 正向查找 xxx ， n/N 进入上/下个匹配。

    ?xxx 是反向

    :s/old/new ：替换当前行的第一个匹配的字符串 old 为 new :s/old/new/g ： 当前行所有 :%s/old/new/g ：全文 :%s/old/new/gc ：全文+询问是否替换

    # 也可以全局匹配。

    $ 跳到行末, ^ 跳到行首。

    配合 CtrlP插件更好。

    CtrlP 插件是直接文件查找，包括对文件内容查找。

    Ctrl+G:：显示当前行。

    :split：将当前文件拆分为上下两个区域

    :new xxx \ :new：新建文件，窗口（默认在上方）

    :vsplit：左右。

    :vnew ：默认在左边。

    r+一个字符ch：当前光标的字符替换成字符ch

    R ：Replace Mode,可以替换多个字符，如果没有字符可以当作 I mode 用。

    c+w/e/$ ：从当前光标开始，删除（到对应motion 的位置）并进入插入模式（相当于修改）

    Shift+q :切换到 Ex mode。

    >> 行首缩进

    << 行首反缩进

    w/W ：下一个单词（MOVE）

    E/e ： 词尾

    shift+d ： 等于 d$.

    ~ : 转换大小写。

    s ; 删除当前字符并进入插入模式

    S: 删除当前行并进入插入模式。

    • In Visual

    y：Copy

    p ：Paste(under Cur)

    P : Paste (over cur)

    d：Cut(and Del)。

    gg,G 功能一样。

    可以利用这个进行全选：ggVG 或者 ggvG$ （注意区分可视和可视行模式）

    • In Visual-Multi

    Ctrl+up/down 多光标

    Ctrl+N 选词，q 取消选择当前词。

    然后跟上Insert Mode 的操作即可。

    git¶

    使用 gitee+git 或 github+git

    首先在当前文件夹下 设置用户名和邮箱：

    1
2
    git config user.name "usrname"
git config user.email "usremail"

    如果需要全局设置可以直接用 --global 参数。

    还需要 remote 远端仓库：

    1
    git remote add origin $url # $url 是你的仓库地址，如果使用 git 协议要在后面加上 .git

    这时候需要鉴权，输入用户名和密码即可。

    连接完之后如果要upload

    1
2
3
4
5
    git add * # 更新Code
git commit -m "Info" # 添加Commit，INFO那里填你需要的Commit。
git push -u origin "master" # master 是要上传的分支。
# 如果曾经写过 `-u` ，那么可以使用 
git push origin "master"
    如果需要覆盖远端仓库： 
    1
2
3
    git add *
git commit -a # 此时会用 vim 打开一个界面确认操作并要求填入 commit。
git push -f --set-upstream origin master:master # 强制覆盖远端的 master 分支

    Supported Vim Versions¶

    Our policy is to support the Vim version that's in the latest LTS of Ubuntu. That's currently Ubuntu 20.04 which contains vim-nox at v8.1.2269.

    Vim must have a working Python 3.6 runtime, compiled with --enable-shared (or --enable-framework). You can check with :py3 import sys; print( sys.version ).

    For Neovim users, our policy is to require the latest released version. Currently, Neovim 0.5.0 is required. Please note that some features are not available in Neovim, and Neovim is not officially supported.

    Supported Compilers¶

    In order to provide the best possible performance and stability, ycmd has updated its code to C++17. This requires a version bump of the minimum supported compilers. The new requirements are:

    Compiler Current Min
    GCC 8
    Clang 7
    MSVC 15.7 (VS 2017)

    YCM requires CMake 3.13 or greater. If your CMake is too old, you may be able to simply pip install --user cmake to get a really new version.

    然后是 Linux 下的安装方式：

    Linux 64-bit¶

    The following assume you're using Ubuntu 20.04.

    Quick start, installing all completers¶

    • Install YCM plugin via [Vundle][]
    • Install CMake, Vim and Python
    1
    apt install build-essential cmake vim-nox python3-dev
    • Install mono-complete, go, node, java and npm
    1
    apt install mono-complete golang nodejs default-jdk npm
    • Compile YCM
    1
2
    cd ~/.vim/bundle/YouCompleteMe
python3 install.py --all
    • For plugging an arbitrary LSP server, check the relevant section

    Explanation for the quick start¶

    These instructions (using install.py) are the quickest way to install YouCompleteMe, however they may not work for everyone. If the following instructions don't work for you, check out the full installation guide.

    Make sure you have a supported version of Vim with Python 3 support, and a supported compiler. The latest LTS of Ubuntu is the minimum platform for simple installation. For earlier releases or other distributions, you may have to do some work to acquire the dependencies.

    If your vim version is too old, you may need to [compile Vim from source][vim-build] (don't worry, it's easy).

    Install YouCompleteMe with [Vundle][].

    Remember: YCM is a plugin with a compiled component. If you update YCM using Vundle and the ycm_core library APIs have changed (happens rarely), YCM will notify you to recompile it. You should then rerun the install process.

    Install development tools, CMake, and Python headers:

    • Fedora-like distributions:
    1
    sudo dnf install cmake gcc-c++ make python3-devel
    • Ubuntu LTS:
    1
    sudo apt install build-essential cmake3 python3-dev

    Compiling YCM with semantic support for C-family languages through clangd:

    1
2
    cd ~/.vim/bundle/YouCompleteMe
python3 install.py --clangd-completer

    Compiling YCM without semantic support for C-family languages:

    1
2
    cd ~/.vim/bundle/YouCompleteMe
python3 install.py

    The following additional language support options are available:

    • C# support: install [Mono][mono-install-linux] and add --cs-completer when calling install.py.
    • Go support: install [Go][go-install] and add --go-completer when calling install.py.
    • JavaScript and TypeScript support: install [Node.js and npm][npm-install] and add --ts-completer when calling install.py.
    • Rust support: add --rust-completer when calling install.py.
    • Java support: install [JDK][jdk-install] and add --java-completer when calling install.py.

    To simply compile with everything enabled, there's a --all flag. So, to install with all language features, ensure xbuild, go, node and npm tools are installed and in your PATH, then simply run:

    1
2
    cd ~/.vim/bundle/YouCompleteMe
python3 install.py --all

    That's it. You're done. Refer to the User Guide section on how to use YCM. Don't forget that if you want the C-family semantic completion engine to work, you will need to provide the compilation flags for your project to YCM. It's all in the User Guide.

    YCM comes with sane defaults for its options, but you still may want to take a look at what's available for configuration. There are a few interesting options that are conservatively turned off by default that you may want to turn on.
