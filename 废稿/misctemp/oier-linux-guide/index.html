    一个 OIer 的 Linux 使用指南

    Preface¶

    我写这篇文章的时候还是一个 OIer，对 \(\texttt{Linux}\) 的了解不算多，但是至少是能很熟练的使用的。

    毕竟有不会的可以看 Documents 学习啊（啃生肉效果会更好）

    使用的系统是 \(\texttt{Ubuntu 20.04.3 LTS}\)，\(\texttt{64 bit}\)，用的是联想的 \(\texttt{X1 Carbon 8GB} + \texttt{Intel Core i5-8250U CPU @1.60GHz}\)

    \(8\) 核。

    另外，感谢 ldrx 学长和 lyl 神仙提供的指导。

    Basic settings¶

    Wi-Fi¶

    这个设置和Windows 没什么区别。

    在你的左下角找到一个有九个点的图标（软件列表），找到设置。

    然后连接即可。

    Livepatch¶

    可以帮你自动更新的东西，并且修复当前已知的 Bug。

    需要 Ubuntu 的单点登陆账号。

    这个在 Ubuntu 的官网注册一个就行。

    https://ubuntu.com/

    然后点击右上角的 Sign In，进去之后选择“我没有 Ubuntu 单点登陆账号”。

    填写信息，用邮箱验证就好了。

    然后在软件列表里面找到 Livepatch

    选择使用 Ubuntu 单点登陆账号登陆。

    之后就可以及时的接收到更新和Bug修复的信息了。

    当然你也可以选择关掉。

    Time¶

    没联网的时候时间可能会有偏差。

    连接网络之后再设置>日期和时间里面选择自动设置日期和时间即可。

    时区选 CST。

    Tips¶

    Windows 和 Linux 有很多区别，在初期使用的时候需要注意。

    使用的时候也有一些小技巧，在这里会一并告诉你。

    • Ctrl+C 在终端当中是强制结束终端运行的进程。

    • Ctrl+Shift+C/V 才是终端当中的复制（\(\texttt{Vim}\) 就又要另外提了，有兴趣可以看一看我写的 Vim Guide）

    • 但是在 GUI （图形化界面）的时候，Ctrl+C/V/X 又可以复制文件

    • 你直接在看见的桌面上只能进行图标的移动，进行复制粘贴操作要在文件管理器当中打开桌面。

    • 如果你在浏览器当中一不小心错误地选择了一个部分，而你发现你在别的地方单击无法去除这个选择的时候，在浏览器的搜索框里面点一下，然后再回来点一下就好了。

    • 在 Linux 当中，以 . 开头的文件和目录会被隐藏，你可以在文件管理器的三根横线那里进行设置。

    • 默认的文本编辑器的名字是 gedit ，但是有的时候没法打开一些文件（无法正常显示），这个时候使用 Vim 就可以了。

    • Vim 可以把行尾序列从 Windows 使用的 [DOS] 改成 Linux 使用的 [Unix] （可以批量把 windows 造的数据变成 Linux 下的格式，也可以倒过来）。

    • 如果你在 USB 当中删除了文件, Linux 会生成一个叫 .Trash000 的目录来保存你的删除文件的信息，如果确定不需要了就直接删除这个文件夹即可。

    • 默认的终端的透明度可以调，在终端的设置>配置文件首选项>（某个配置文件的名字，默认是未命名）>颜色 里有一栏是

    Use Transparent background ，可以用来调整，如果没法调整，先把下面的 Usetransparency for system theme 关闭之后再打开 Use Transparent background 即可。

    • 如果你有了 Ubuntu 的单点登陆账号，那么你就可以使用 Ubuntu 的云剪贴板：https://pastebin.ubuntu.com/

    Commands¶

    Conventions¶

    <Command> 表示一个命令。

    -[Option] 或 --[Option] 表示一个选项。

    # xxxx 表示一个注释。

    sudo¶

    准确的说，这应该不算是一个；命令？

    感觉更类似一个 “声明”，也就是声明后面的命令是以 root 权限运行。

    sudo <Command>

    在当前终端下使用，第一次需要输入密码，第二次就不需要了。

    在 /usr /bin 这种系统目录底下一般都需要。

    在用 apt dpkg 这种软件包管理器进行操作的时候一般也需要。

    cd¶

    用来切换目录。

    常见的几种变形：

    1
2
3
4
    cd ~/xxx/yyy/zzz # 切换到 ~/xxx/yyy/zzz 这个目录
cd ~ # 切换到主目录
cd   # 功能同上
cd .. # 回到上一层目录

    cat¶

    直接查看某个文件

    1
2
3
    cat qwq.cpp # 查看 qwq.cpp
cat ~/code/qwq.cpp  # 如果你不在那个文件的目录就这么写
cd ~/code/qwq && cat qwq.cpp # 功能同上

    diff¶

    比较两个文件的差异

    1
    diff qwq.out qwq.ans # 比较这两个文件的差异

    rm¶

    删除某个文件

    1
2
3
4
5
6
    rm -[Option] DictName/FileName
# Option 参数如下：
# -i 逐一询问
# -f 不管是不是只读，全部删除干净
# -r 将目录和他里面的所有文件和目录都删除（删除目录需要写上）
# <void> （不填） 不询问直接删除（无法在回收站找回）

    mv¶

    移动某个文件/目录 或者用来改名

    1
2
3
4
5
6
7
    mv -[Option] foo bar
# Option 参数列表
# -b 当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份
# -i 如果指定移动的源目录或文件与目标的目录或文件同名，则会先询问是否覆盖旧文件
# -f 不会询问，直接覆盖旧文件
# -n 不要覆盖任何已存在的
# -u 当源文件比目标文件新或者目标文件不存在时，才执行移动操作

    1
2
3
4
    mv file1 file2 # 将 file1 改名为 file2
mv file dict # 将 file 移动到 dict 目录
mv dict1 dict2 # 如果 dict2 已经存在，把 dict1 移动到 dict2
               # 否则把 dict1 重命名为 dict2

    cp¶

    复制某个文件/目录

    1
2
    cp qwq.cpp ~/code # 把 qwq.cpp 复制到 ~/code 目录下
cp -r qwq/ code # 把 qwq/ 目录下的所有文件复制到 code 目录下 （不要忘记 -r）

    ls¶

    列出当前目录下的所有文件/目录

    1
2
    ls # List all
ls | grep Touhou # 列出当前目录下所有含有关键字 Touhou 的文件/目录

    mkdir¶

    新建一个文件夹。

    1
2
    mkdir qwq # 在工作目录下新建一个目录 qwq
mkdir qwq/qaq # 在工作目录下新建 qwq/qaq ，并且如果 qwq 不存在会自动建立，如果qwq不存在但是没有 -p，会报错。

    exit¶

    退出终端

    1
    exit

    alias¶

    设置一个自定义命令

    1
    alias la = ls -all

    apt¶

    软件包管理器。

    下载之前建议先换源。

    在 >全部软件>软件和更新>Ubuntu软件>下载自当中选择并重启。

    建议选择清华源（较为稳定，速度给力） http://mirrors.tuna.tsinghua.edu.cn/ubuntu

    1
2
3
4
5
6
    sudo apt install qwq # 安装 qwq
sudo apt -y install qwq # 安装过程全部 yes
sudo apt update # 列出所有的可以更新的软件包
sudo apt upgrade # 升级软件包
sudo apt install qwq qaq ovo # 安装 qwq,qaq,ovo
sudo apt remove qwq # 删除 qwq

    dpkg¶

    一个 debian 系的软件包管理器。

    常用的也就三个

    1
2
3
    sudo dpkg -i qwq.deb # 解压 qwq.deb
sudo dpkg -l # 列出已经安装的软件包 ，可以配合 | grep xxx 快速查找对应软件包
sudo dpkg --remove qwq # 卸载 qwq

    Wine¶

    wine-origin¶

    这是一个帮助你在 Linux 系统上运行 Windows 软件的环境

    下载：

    1
    sudo apt install wine

    使用：

    1
    wine qwq.exe

    第一次使用的时候会弹出一个界面，提示你 wine 正在初始化配置，这个过程一般需要两分钟。

    注意不要把终端关掉了，不然你的 wine 也会停止工作。

    deepin-wine-for-ubuntu¶

    国内的 deepin 公司做了很多努力，在 wine 的基础上移植了 qq,wechat,TIM 这些软件。

    （他们毕竟需要 setup,而原版 wine 没法利用 setup.exe 安装）

    然后又有两个大神分别把这个移植到了 Ubuntu 上。

    其中一个的项目地址：https://github.com/zq1997/deepin-wine

    屏幕缩放，图片，字体的问题都在这个项目的 README.MD 当中。

    winetricks¶

    用来给 wine 打一些运行库的补丁。

    比如 vsrun2005, dxvk 这种

    首先

    1
    sudo apt install winetricks

    下载完之后直接

    1
    winetricks [PackageName]

    就可以打补丁了。

    建议打上这几个：

    • dxvk
    • directx9
    • vsrun2005
    • vsrun2009

    可以解决大部分的 wine 无法打开程序的问题。

    因为 Ubuntu 20.04 系统基于 Gnome 桌面，所以我们可以利用插件进行美化。

    Zsh¶

    这个东西号称 Linux 下的最强 Shell（确实）。

    功能强大：

    1. 自动补全
    2. 历史记录
    3. 自定义
    4. 语法高亮
    5. 自定义主题
    6. 插件支持
    7. ………………

    用它肯定会减少你输入的时间，提升你的效率。

    Install¶

    1
2
    sudo apt-get install zsh
chsh -s /bin/zsh #设置为默认Shell

    然后重启。

cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

    配置部分可以参考：https://zhuanlan.zhihu.com/p/62501175
