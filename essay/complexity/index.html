    时间复杂度分析

    理解方式比较糙，不过现在这个阶段勉强能用就行。

    如果要之后细致的研究，那就之后再说吧。

    基本操作数¶

    计算机进行一次类似赋值，四则运算，访问变量的操作可以看作一次基本操作。

    不同的 PC 的性能不同，就会导致执行基本操作的速度不同。

    平常说有的机子跑得快，一秒能跑 3e9，有的机子跑得慢，一秒只能跑 8e7。

    这里的 3e9 和 8e7 就是执行基本操作的次数，也可以简单的理解为计算量。

    一般对算法效率的衡量，都是利用基本操作的计数进行的。

    算法复杂度¶

    一般记作 \(T(n)\)，表示数据规模为 \(n\) 时，这部分算法的效率。

    这个东西可以简单的理解为，自变量为 \(n\) ，函数值为数据规模为 \(n\) 时的总计算量的函数，当然这里的“自变量”可以是很多个，取决于你的程序的输入规模。

    大 O 记号¶

    用来表示一个函数的渐近上界，一般在算法复杂度分析时都使用大 \(O\) 记号。

    可以简单的理解为：\(f(n) = O(g(n))\)，则 \(g(n)\) 就是 \(f(n)\) 的一个上界。

    比如 \(f(n) = n/2\)，那么 \(f(n) = O(n)\)。

    一些常见的复杂度：

    • 线段树：单次操作 \(O(\log n)\)

    • 双指针：\(O(n)\)

    • 冒泡排序：\(O(n^2)\)

    • 高维前缀和：\(O(n\times 2^n)\)

    • 四则运算：\(O(1)\)。

    • 快速排序：平均 \(O(n \log n)\)，最坏 \(O(n^2)\)。

    • std::vector<_Tp>::push_back()：均摊 \(O(1)\)。

    一些注意：

    1. 使用大 \(O\) 记号的时候，加法常数项可以忽略不计，因为当输入规模趋近于正无穷时，加法常数项对于算法的运行效率的影响是微乎其微的。

    2. 除去最高次项，其它次项可以忽略不计，理由同上。

    3. 最高此项乘上的常数可以不写，理由同上。

    不过这种我们一般都会在讨论的时候说，某某某操作带有 \(3,14\) 大小的常数之类的。

    但是像 \(\dfrac{1}{32},100\) 这种对效率影响比较大的常数还是要写上的。

    1. 复杂度里面的 \(\log\) 统一视为 \(\log_2\)。由换底公式可以得到，任何对数函数无论底数是多少，都具有相同的增长率，所以底数一般省略不写。

    两个法则：

    1. 加法法则：\(T1(n) + T2(n) = O(\max(T1(n),T2(n))\)

    2. 乘法法则：\(T1(n)\times T2(n) = O(T1(n)\times T2(n))\)

    均摊复杂度¶

    这个东西本质上和平均复杂度是一样的，只不过侧重点不同。

    平均复杂度强调最后的总复杂度，均摊侧重于操作。

    比如 std::vector<_Tp>::push_back()

    vector 的原理是，每次数组满了之后扩容一倍，扩容的复杂度是 \(O(n)\) 的。

    然后直接把元素放进去的复杂度是 \(O(1)\) 的。

    看起来每次操作是 \(O(n)\) 的，但实际上，只有满了的时候才会扩容。

    并且每次扩容并不是都要扩容 \(n\) 那么长，这里 \(n\) 只是一个渐近上界。

    实际上，他们扩容的总长度应该是 \(1 + 2 + 4 + 8 + \dots + n\) （假设 \(n\) 是 \(2^k\) 的形式）

    把 \(n\) 看作单位长度 \(1\)，那么这玩意儿就是 \(\frac{1}{1} + \frac{1}{2} + \frac{1}{4} + \dots + \frac{1}{n}\)，一个经典结论就是，当 \(n \to +\infty\) 的时候，这个式子无限接近于 \(2\)。

    （另外一个结论是 \(\frac{1}{1} + \frac{1}{2} + \frac{1}{3} + \dots + \frac{1}{+\infty} = n\ln n\)）

    所以扩容的总长度是 \(2n\)，而每放进一个元素的花费是 \(O(1)\) 的，总共花费 \(T(n) = 3n = O(n)\)。

    那么均摊下来，每一次操作的复杂度就是 \(O(1)\) 的。

    这种方式被称作均摊分析。

    最坏复杂度¶

    因为我们一般只考虑渐近上界，所以一般只讨论最坏复杂度。

    比如快速排序，它的平均复杂度是 \(O(n\log n)\)。

    但是可以用某些方式把它卡到 \(O(n^2)\)，并且不可能再高了。

    这时候就称它的最坏复杂度是 \(O(n^2)\)。

for(int i = 1; i <= n; ++i) {
    for(int j = 1; j <= n; ++j) {
        c[i + j - 1] = a[i] + b[j];
    }
    for(int k = 1; k <= n; ++k) {
        c[i + k - 1] *= k;
    }
}

    根据两个法则，这个程序的复杂度 \(T(n)\) 应当等于 \(T_i(n) \times \max(T_j(n),T_k(n))\)。

    所以 \(T(n) = O(n^2)\)。

    // e 储存了一个 n 个点 m 条边的无向连通图
void dfs(int u) {
    vis[u] = true;
    for(int i = head[u]; ~i; i = e[i].Next) {
        int v = e[i].ver;
        if(vis[v]) continue;
        dfs(v);
    }
}

    因为有 vis 数组和 continue 的存在。

    这个 dfs 只会遍历每个点和每条边一次。

    所以总复杂度为 \(O(n + m)\)。

    一些要注意的点¶

    不要看到几个 for 循环就直接下定论，要仔细分析每层循环的实际计算量。

    将算法分成几个块，然后利用两个法则分析。

    特别要注意的是记忆化数组，continue,break 语句这类存在。

    有很多常用函数的复杂度和常数一定要了解。

    比如 sort, lower_bound, log 这些函数。

    翻 cpp ref 是最好的选择。
