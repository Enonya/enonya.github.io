    解决的最好方案¶

    打开 -Wall，打代码的时候稳一点。

    多花时间读题，防止看错题导致浪费时间以及心态爆炸。

    你多花 5min 读题可比你 1h 都困惑条件好。

    在草稿纸上理清楚思路再开始写，有些地方加上注释说明用途，方便调试。

    收卷之前10min检查自己的文件，freopen，路径，格式。

    最好打开自动保存防止代码丢失

    保留一些BF，有问题的正解，并标记，方便唤起新的思路并提醒自己，说不定过一会看一看之前的Code就发现错误了呢？

    没写出来不要心态爆炸，稳重出省一。

    和 OPJ 写法一样的拼音变量虽然很 low，但是debug却很实用。（怪不得OPJ的debug那么厉害）

    或者有多个类似变量的时候用下划线+功能标记一下。

    把你已经拿到的部分分利用 if 框起来，防止你写的伪正解，覆盖了你本来能拿到的分。

    做了总比不做好，并且保持思维的清晰有序。（来自 Benq）

    容易手滑写错（漏）的¶

    1. 当你遍历链式前向星的时候：

      for(int i=e[u].head;i;i=e[u].Next)

      把 \(u\) 和 \(i\) 搞混

    2. 当有多层循环的时候：

      1
2
      for(int i=1;i<=n;++i) 
    for(int j=1;j<=m;++i)

      在该写 ++j 的时候写 ++i

    3. 当你读入一个 \(n \times m\) 的矩阵的时候： 

      1
2
      for(i;1->n)
    for(j;1->n)

    4. for(int i=1;i<=n;--i)

    5. printf 写了一个 &。

    6. int a[siz][0];

    7. mod 0

    8. printf("%d",x/0);

    9. 要求取 \(\min\) 却res=INT_MIN ,要求 \(\max\) 却 res=INT_MAX。

    10. 位运算不知道运算优先级的情况下不打括号。

    11. for(int i=1;i<=strlen(s);++i) （多次调用 strlen 导致TLE）

    12. return gcd(b,b%a) （把欧几里得的结论背反）。

    13. 写三目运算符的时候以为不满足条件才会执行 : 前面的。

      比如 min(int &a,int b){ return a<b?a=b:a;}:

    14. 把 int 记成 2 个bit ，long long 记成 4 个 bit ，导致空间算错。

    15. 主函数的最后压行却写：return printf("%lld\n",res);，直接 return了 printf的返回值，导致RE。

    16. 想要取出 \(s\) 的 \([0,i-1]\) ，却写了 s.substr(0,i-1)。

      (忘记了 substr 的第二位是说取出来的这个子串的长度)。

    17. int p[si];p[0]=1; 和 int p[si]={1};前者 \(\text{O}(1)\) ，后者 \(\text{O}(n)\)

    18. 树链剖分的时候： dfs2(hx[i][i]); （树 点 剖 分）

    19. 线段树区间加的时候，pushdown 里用 =，区间修又去用 +=。（没搞清楚到底要干什么）

    20. 不要写 a[++cnt]=b[cnt]=d; （会UB）

    21. long long n;scanf("%d",&n); （格式控制符写错）（很容易查不出来）

    22. unsigned long long qwq; 然后 printf("%ull",qwq); （实际上是 %llu）

    23. 搞反数组的维度分别代表什么。

    24. 单调队列的 head 和 tail 写反。

    25. UB：tmp[i]++=cnt[j]--

    26. 名字比较像的变量混着使用。

    27. 比较大的数用一些浮点数返回值函数计算的时候忘记强制转成 ll 导致输出科学计数法。

      比如 pow(3,19) 这种很大的，都要强制转换。

    28. 一个例子：

      1
2
3
4
5
6
      struct Edge{
    int head,ver,Next;
    inline void init(){ head=-1; }
}e[si_m<<1];
e->init() // 这样写只会清空 e[0] 而不是所有的。
    // STL 能这样写，实际上是因为 STL 重载了 [] 所以让它看起来类似一个数组。

    29. 少写 std::bitset<int>b[si]; b->reset()，常数很大。

    30. 倍增的时候建议先循环第二维，因为要小心第一位不是按照正常顺序转移导致出现后效性。

    31. 有除法且 1.0 × 某个数的时候要用括号括起来，不然会 /1.0 然后乘上原来要除的数。

    容易忘记（写，算，查）的细节¶

    1. 多组数据不清空型

    2. 图论题不清 cnt，head

    3. 数组不清 0

    4. vector 不 clear()

    5. 线段树 tag() 不清零。

    6. 用 for(1->n) 清零，init 却写在读入 n 的前面。

    7. 上面这个用构造函数的时候也是同理。

      所以如果有构造函数建议单独定义出来而不是跟在 struct 之后。

    8. 段错误（RE）型

      • 数组开小
      • 线段树四倍空间
      • 树的两倍空间
      • 无向图的两倍空间
    9. scanf() 不是读字符数组但是不写 &
    10. long long
    11. 负数考虑了么？
    12. 最简单的矩阵类型dfs不写continue 和 return （没写边界）

    13. /0 ，erase(end())。

    14. queue 和 stack 不写 pop

    15. operator 不写 const

    16. vector 的 size() 不写 (int) 然后就减了一个数。

    17. 你写线段树区间操作在 update，query 里面 pushdown 了吗？

    18. bitset<si>vis[si]; ... vis.reset() （应该用 -> reset()，但是常数大，建议 for+.reset()）

    19. 写树形DP的时候不写 vis 或者 if(v==fa) continue;

    20. 倍增LCA，dep[1]没设为1，dfs的时候忘记 dep[v]=dep[u]+1

    21. 自环和重边判了吗？

    22. 到底是 scanf("%d%d%d",&n,&m,&q); 还是scanf("%d%d%d",&m,&n,&q);还是scanf("%d%d%d",&q,&n,&m); ？？（注意读题的输入格式）。

    23. 当变量名特别多，比如输入里有个 \(k\) ，循环又有三层的时候：

    24. 1ll 和 1.0乘了吗？

    25. 有没有考虑系统函数的精度导致的错误？

      比如：当你需要对一个在 \(2^{32}\) 以上的数算 \(\log_2\) 的时候，精度很容易炸，建议手写：

      还有浮点数二分的时候，有没有考虑 \(eps\) 带来的误差？

      这里建议算好次数，手动循环。

    26. 统计答案忘记取模。

    27. 分类讨论题不考虑像 \(n=1\) 之类的特殊解导致的做法出错。

    28. 在严格匹配输出的时候多输出一个空格，多输出一个换行。

    29. 算概率忘记样本空间，算期望忘记变量取值，期望DP正着推却没处理好后效性问题。

    30. Dp 没初始化，少转移，没考虑后效性，过了大样例直接跑路。

    31. unsigned long long x=1-2;unsigned long long y=abs(x);（忘记 unsigned 不带符号）。

    32. 写了个函数但是你没调用。

    33. 写的 int 类型的函数但是你没写返回值。

    34. const int mod=99824353 （凉心出题人的坑模数1）

    35. const int mod=998244533 （凉心出题人的坑模数2）

    cin>>n;

    37. 在数据很大的时候一定不要使用 % ，直接手写即可。

    38. 取模的时候如果里面可能有负数要

      先膜再加再膜:((a+b)%mod+mod)%mod

      如果没有负数就 (a+b)%mod

    39. 库函数自带的 max，min 要求两个参数变量类型相同。

    40. heap-dijkstra 不写 -dis[v]（贪心变谦虚）

    容易因为文件，系统和环境出错的¶

    1. int time,y1; （Linux 下的关键字）
    2. Linux\POJ 下使用 INT_MAX 但不写 #include<climits>（万能头的话，再见）
