    倍增

    倍增¶

    用成倍增长和二进制拆分的思想优化类似递推的方法。

    常见的应用是 LCA 和 ST table。

    推导的柿子是 \(to[i][j] = to[to[i][j-1]][j-1]\)。

    其中 \(to[i][j]\) 表示 \(i\) 这个元素经过 \(2^j\) 次操作得到的什么东西。

    使用的时候可以把状态空间关于 \(2\) 的正整数次幂有可划分性的类递推问题从 \(\text{O}(n)\) 优化到 \(\text{O}(\log n)\)。

    有可划分性的一种表现是：\(to[i][j] = to[ to[ to[ to[i][j-3] ][j-3] ][j-2] ][j-1]\)，并且这样可以一直嵌套下去。

    然后一般对于一个数 \(k\) ，要想知道 \(i\) 经过 \(k\) 次操作的结果是什么。

    利用所有正整数都可以表示为若干个 \(2\) 的次幂项的和这个性质。

    先求出 \(to\)。

    然后把 \(k\) 转成二进制，对它进行二进制拆分。

    具体来说，从大往小 枚举 \(k\) 在二进制下的每一位，如果当前位 \(j\) 是 \(1\) ，那么使 \(ans = to[ans][j]\)。

    然后这样不断跳，就可以得到答案。

    推荐求 \(to\) 的时候把 \(j\) 放在外面，因为如果所有的 \(i\) 依次组成的不是一个单调的序列，转移的时候可能出现从 \(to[3] \to to[1]\) 这种。

    那么把 \(i\) 放在外面就会有问题，如果 \(i\) 是单调的，那么怎样都行。

        lg = (int)(log(k) / log(2)) + 1; 
    for(int i = 1; i <= n; ++i) t[i][0] = p[i];
    for(int j = 1; j <= lg; ++j) {
        for(int i = 1; i <= n; ++i) {
            t[i][j] = t[t[i][j - 1]][j - 1];
        }
    }
    // 这里如果 i 在外面会导致出现 t[i][j] = 0。
    // 因为这一题的 i 不是按照升序排列来 DP 的，如果 i 在外面就有后效性。
    for(int i = 1; i <= n; ++i) dest[i] = i;
    // dest[i] 最终是经过 k 轮, 原来在 i 的人所在的位置。
    for(int j = lg; j >= 0; --j) {
        if(k >> j & 1) {
            for(int i = 1; i <= n; ++i) {
                dest[i] = t[dest[i]][j];
                if(t[dest[i]][j] == 0) cerr << "Error on #" << i << ", " << j << endl;
            }
        } // 二进制拆分
    }

    倍增推式子的时候都要遵循这样的一个规则：

    \(u \sim f_{u,i}\) 这一段的信息要由 \(u \sim f_{u, i -1}\) 和 \(f_{u,i - 1} \sim f_{f_{u,i-1},i-1}\) 合并来得到。

    （\(f_{f_{u,i-1},i-1}\) 就是 \(f_{u,i}\)）

    然后初始化的时候只需要初始化 \(f_{u,0}\)，然后在递推式子的时候先正序循环，保证从小到大更新。

    二进制拆分（或者说：“拼凑”） 的时候再使用倒序循环。

    可以类比线段树的 pushup 函数。
